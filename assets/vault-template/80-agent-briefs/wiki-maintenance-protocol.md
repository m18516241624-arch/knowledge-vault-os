# Wiki Maintenance Protocol

Maintain the vault as an LLM-compiled Markdown wiki, not as a dumping ground.

## Four Layers

1. Source governance: registered sources and sensitivity rules.
2. Evidence: inbox, source packets, capture ledgers, raw exports.
3. Knowledge compilation: projects, decisions, methods, context, skills.
4. Agent reading: global index and agent briefs.

## Maintenance Flow

1. Scan new evidence and recent changes.
2. Classify source level, sensitivity, and value.
3. Update capture ledger when a source matters.
4. Promote stable conclusions to the right topic pages.
5. Update agent briefs only when future agents need the rule.
6. Run a light lint for missing metadata, stale pages, duplicates, and sensitive-boundary risks.

## Success Standard

A future agent should understand what matters without rereading raw evidence unless traceability is needed.
