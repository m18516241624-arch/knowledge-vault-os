# v0.2 Scope

## Goal

Close the biggest gap between the public skeleton and a running local knowledge system without bundling private data or account connectors.

## Included

- v0.1 plugin manifest, Skill, references, vault template, install, package, and validation flow.
- `daily-capture`: scans local source folders for changed files in a target date window and writes source packet, capture ledger, and daily log entries.
- `compile-inbox`: promotes only source packets explicitly marked `decision: compile`, with stable claims and a safe promotion target.
- `dashboard-lite`: builds a local JSON index and renders a static dashboard with distribution, project cards, evidence queue, recent notes, and health issues.
- Extended validation covering manifest, Skill frontmatter, init, scan, daily capture, compile inbox, dashboard build, lint, and public-safety scan.
- Release hardening for repeated runs: daily capture writes unique source packets, and compile-inbox skips source packets already compiled into the target page.
- Dashboard Lite browser screenshot verification for desktop and mobile viewports.
- GitHub validation workflow and changelog for public release hygiene.

## Not Included Yet

- Deep semantic summarization of file contents.
- Codex session parsing.
- Account connectors.
- Chat platform ingestion.
- Cloud sync.
- Vector search.
- Team permissions.
- Fully automated topic-page writing for ambiguous evidence.

## Product Boundary

v0.2 is a local-first operating workflow. It gives Codex and the user a safe structure for evidence and promotion, but it does not pretend deterministic scripts can replace human or LLM judgment for sensitive semantic compilation.
