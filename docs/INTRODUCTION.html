Local-first knowledge workflow for Codex

Knowledge Vault OS

一套可安装、可复用的 Codex 知识库操作系统，把信息从感知、证据、编译，一路推进到 Agent 可调用的长期 Markdown 主库。

打开 GitHub 仓库 下载 v0.2.0

它解决什么问题

不是临时问答

它不把资料直接丢给 RAG，而是先进入证据层，再判断哪些结论值得长期保存。

不是普通模板

它包含 Codex Skill、vault 模板、本地脚本、dashboard-lite 和发布验证链路。

不是云端黑盒

Markdown/Obsidian 是唯一长期主库，敏感资料默认留在本地和证据层。

核心工作流

感知扫描项目、文件、笔记或外部来源。
来源登记记录来源、时间窗口、敏感等级。
证据入账先写入 source packet 和 capture ledger。
分块摘要拆出候选结论，保留可追溯路径。
晋升判断只晋升稳定、可复用、已脱敏内容。
主题页编译写入项目、决策、方法和 Agent Brief。
Agent 调用Codex 做事前读取压缩上下文。

包里有什么

Codex Skill

负责初始化、接入、编译、调用和维护知识库。

Markdown Vault 模板

包含 inbox、projects、decisions、methods、agent briefs 和维护目录。

本地脚本

支持 daily capture、compile-inbox、dashboard build、lint、安装、打包和验证。

Dashboard Lite

用静态页面查看目录分布、项目状态、证据队列、近期笔记和健康问题。

隐私边界

真实聊天、公司资料、私有索引和机器路径不进入公开包。

发布验证

GitHub Actions 会跑 validate、dashboard 截图验收和发布包构建。

快速开始

git clone https://github.com/m18516241624-arch/knowledge-vault-os.git
cd knowledge-vault-os
npm install
npm run validate
node scripts/init-vault.mjs ~/Documents/MyKnowledgeVault

适合谁

长期使用 Codex 的人

希望 Codex 不只是回答问题，而是能持续读懂自己的项目和方法。

重视本地知识主权的人

希望 Obsidian/Markdown 留作事实源，而不是把私密资料交给外部系统。

想产品化知识流程的人

希望把自己的工作方式打包给别人安装，而不是只给一份笔记模板。

当前边界

v0.2.0 不包含账号连接器、云同步、向量数据库、自动聊天记录抓取或团队权限系统。它先把本地优先、证据先行、谨慎编译和 Agent 调用这条主链跑通。
